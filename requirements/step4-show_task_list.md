# Step 4: 📚 タスク一覧機能

## 要件

サインイン中のユーザー自身が作成したタスクと、その他のユーザーが作成したタスクを一覧表示する

タスクが多い場合は、1回に最大15件ずつ追加で読み込めるようにする

## 詳細

### Rails

- 次のエンドポイントを作成する

| HTTP メソッド | パス | アクション名 | 用途 |
| --- | --- | --- | --- |
| `GET` | `/api/tasks` | `index` | タスク一覧を取得する |

- コントローラー名は `TasksController` とする
- `index` では Rails session に保存されているユーザー識別子から、サインイン中のユーザーを取得する
- クエリパラメーター `type` によって、取得するタスクを切り替える
  - `my`: サインイン中のユーザーが作成したタスクを取得する
  - `others`: サインイン中のユーザー以外が作成したタスクを取得する
  - `type` が省略された場合は `my` として扱う
- クエリパラメーター `page` によって、取得するページを指定する
  - `page` は1から始まる整数とする
  - `page` が省略された場合は `1` として扱う
- タスクは1ページあたり最大15件取得する
- タスクは作成日時の新しい順に並べる
  - 作成日時が同じ場合は `tasks.id` の大きい順に並べる
- 各タスクと、そのタスクを作成したユーザーの情報を返す
- 一覧取得時に **N + 1問題** が発生しないようにする
- 次のページに表示できるタスクがある場合は `has_next` を `true` とする
- 以下の形式の JSON を HTTP ステータス 200 で返す

```json
{
  "tasks": [
    {
      "id": 1,
      "body": "タスク内容",
      "is_completed": false,
      "created_at": "2026-07-15T10:00:00.000+09:00",
      "user": {
        "id": 1,
        "name": "ユーザー名"
      }
    }
  ],
  "has_next": true
}
```

### Vue

- Step 3 で作成したホームページにタスク一覧を追加する
- ページの初回表示時に `GET /api/tasks?type=my&page=1` を呼び出す
- 表示するタスクを切り替える「My タスク」と「みんなのタスク」を用意する
  - 「My タスク」では `type=my` を指定する
  - 「みんなのタスク」では `type=others` を指定する
  - 表示する種類を切り替えた場合は、表示中のタスクとページ番号をリセットし、1ページ目から取得し直す
- 各タスクに、タスクを作成したユーザー名、タスク内容、作成日時を表示する
- タスク内容は Vue のテキスト展開を使って表示する
  - **XSS** を防ぐため、タスク内容の表示に `v-html` を使わない
- API の `has_next` が `true` の場合だけ「さらに読み込む」ボタンを表示する
  - ボタンを押したら次のページを取得する
  - 取得したタスクは、現在表示している一覧の末尾に追加する
  - API を呼び出している間は、ボタンを無効にして重複呼び出しを防ぐ
- Step 3 のタスク作成に成功した場合は、「My タスク」の1ページ目に切り替えて一覧を取得し直す
- API の呼び出し処理は、既存の repository 層にタスク一覧取得用の処理を追加して行う

### エラーハンドリング

- サインインしていない場合は HTTP ステータス 401 を返す
- `type` が `my` または `others` 以外の場合は HTTP ステータス 400 を返す
- `page` が1未満、または整数でない場合は HTTP ステータス 400 を返す
- 対象のタスクが存在しない場合はエラーにせず、HTTP ステータス 200 で空の `tasks` 配列と `has_next: false` を返す

```json
{
  "tasks": [],
  "has_next": false
}
```

- エラー時は、次のようにエラー内容を JSON で返す

```json
{ "error": "エラー内容" }
```

- Vue 側でタスク一覧の取得に失敗した場合は、`window.alert` 関数を使ってエラー内容をユーザーに知らせる

### データベース

本要件では新しいテーブルやカラムを追加しない

Step 3 で作成した `tasks` テーブルと、`users` テーブルを使用する

## 動作確認

- 「My タスク」には、サインイン中のユーザーが作成したタスクだけが新しい順で表示される
- 「みんなのタスク」には、サインイン中のユーザー以外が作成したタスクだけが表示される
- 各タスクにユーザー名、タスク内容、作成日時が表示される
- 対象のタスクが16件以上ある場合、初回は15件まで表示され、「さらに読み込む」で残りのタスクが追加される
- 次のページがない場合と、タスクが1件もない場合は「さらに読み込む」が表示されない
- 一覧の種類を切り替えたとき、切り替え前のタスクが混ざらない
- タスク作成後、作成したタスクが「My タスク」の先頭に表示される
- HTML タグや JavaScript を含むタスク内容が、HTML やスクリプトとして実行されない
