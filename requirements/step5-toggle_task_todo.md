# Step 5: ✅ タスク完了化 / 未完了化機能

## 要件

サインイン中のユーザー自身が作成したタスクを完了または未完了に変更できるようにする

また、タスク一覧を「すべて」「未完了」「完了」で絞り込めるようにする

## 詳細

### Rails

- 次のエンドポイントを作成する

| HTTP メソッド | パス                             | アクション名 | 用途                 |
| ------------- | -------------------------------- | ------------ | -------------------- |
| `PUT`         | `/api/tasks/:task_id/complete`   | `complete`   | タスクを完了にする   |
| `PUT`         | `/api/tasks/:task_id/incomplete` | `incomplete` | タスクを未完了にする |

- `complete` では対象の `tasks.is_completed` を `true` に更新する
- `incomplete` では対象の `tasks.is_completed` を `false` に更新する
- サインイン中のユーザーが作成したタスクだけを更新できるようにする
  - リクエストから `user_id` を受け取ったり、指定したりしてはいけない
- すでに同じ完了状態のタスクへ同じリクエストを送った場合もエラーにしない
- 更新に成功した場合は、以下の形式の JSON を HTTP ステータス 200 で返す

```json
{
  "task": {
    "id": 1,
    "is_completed": true
  }
}
```

#### タスク一覧の絞り込み

- Step 4 で作成した `GET /api/tasks` にクエリパラメーター `completion` を追加する
  - `all`: 完了状態にかかわらずすべてのタスクを取得する
  - `incomplete`: `is_completed` が `false` のタスクを取得する
  - `complete`: `is_completed` が `true` のタスクを取得する
  - `completion` が省略された場合は `all` として扱う
- `type`、`completion`、`page` を併用してタスクを取得できるようにする
- 絞り込み後のタスクに対して、Step 4 の並び順と15件ずつのページングを適用する

### Vue

- タスク一覧に「すべて」「未完了」「完了」を選択できるセレクトボックスを追加する
  - 初期値は「すべて」とする
  - 選択を変更した場合は、表示中のタスクとページ番号をリセットし、1ページ目から取得し直す
- My タスクには完了状態を変更するチェックボックスを表示する
  - チェックをつけた場合は `PUT /api/tasks/:task_id/complete` を呼び出す
  - チェックを外した場合は `PUT /api/tasks/:task_id/incomplete` を呼び出す
  - API を呼び出している間は対象のチェックボックスを無効にする
  - 更新に成功したら、一覧全体を再取得せずに対象タスクの完了状態を更新する
  - 絞り込みの対象外の状態に変更した場合も、次に一覧を取得し直すまでは現在の一覧に残す
- みんなのタスクでは、チェックボックスの代わりに「完了」または「未完了」の状態を表示する
- API の呼び出し処理は、既存の repository 層に完了状態の取得・更新処理を追加して行う
- API の呼び出しに失敗した場合は、変更前の状態に戻し、`window.alert` 関数でエラー内容をユーザーに知らせる

### エラーハンドリング

- サインインしていない場合は HTTP ステータス 401 を返す
- 対象のタスクが存在しない場合は HTTP ステータス 404 を返す
- 他のユーザーが作成したタスクを更新しようとした場合も HTTP ステータス 404 を返す
- `completion` が `all`、`incomplete`、`complete` 以外の場合は HTTP ステータス 400 を返す
- エラー時は、次のようにエラー内容を JSON で返す

```json
{ "error": "エラー内容" }
```

### データベース

本要件では新しいテーブルやカラムを追加しない

Step 3 で作成した `tasks.is_completed` を使用する

## 動作確認

- My タスクのチェックボックスで完了・未完了を変更できる
- みんなのタスクでは完了状態を確認できるが、変更はできない
- 同じ完了状態へ繰り返し更新してもエラーにならない
- 「すべて」「未完了」「完了」でタスク一覧を絞り込める
- 完了状態の絞り込みが My タスク・みんなのタスクと15件ずつの追加読み込みの両方と併用できる
- 更新中のタスクだけが操作不可になる
- 更新に失敗した場合は画面表示が元の状態に戻る
